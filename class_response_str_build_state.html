◆ buildContentTypeHeader()

void ResponseStrBuildState::buildContentTypeHeader ( )

レスポンスヘッダーに適切なContent-Typeを設定します。

CGIからContent-Typeが返された場合、それをレスポンスヘッダーに使用します。 そうでない場合は、リクエストされたリソースの拡張子に基づいてContent-Typeを決定します。

Note
  • 拡張子が見つからない場合やMimeTypeマップに存在しない場合は、デフォルトのContent-Typeが設定されます。

Note
  • ステータスラインのフォーマットは HTTP-version SP status-code SP reason-phrase CRLF です。
  • ABNF
    status-line = HTTP-version SP status-code SP reason-phrase CRLF; ex.HTTP1.1
    status-code = 3DIGIT; ex.200
    reason-phrase = *( HTAB / SP / VCHAR / obs-text ); ex.OK

bool ResponseStrBuildState::shouldCloseSocket ( )

クライアントのリクエストヘッダーに "Connection: close" が存在するか確認します。

この関数は、_socketStatesに格納されたソケットヘッダーを調べることで、クライアントのリクエストに"Connection: close" ヘッダーが存在するかどうかを確認します。

Parameters
