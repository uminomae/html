◆ ValidationRequestInLocation()

void IdentifyLocationState::ValidationRequestInLocation ( )
protected

現在のリクエストが特定のロケーションディレクティブに適合しているか検証します。

この関数は、現在のHTTPリクエストが接続されたロケーションディレクティブの条件に合致するか検証し、条件に合致しない場合に適切なステータスコードをHttpTransactionに設定します。 次のような検証を行います：

  • リクエストヘッダーのContent-Lengthがロケーションの最大ボディサイズを超えていないか（413 Payload Too Large）。
  • リクエストメソッドがロケーションが許可するHTTPメソッドに含まれているか（405 Method Not Allowed）。
  • ロケーションが永続的に移動しているか（301 Moved Permanently）。
Note
この関数は、リクエストが条件に合致しない場合にのみステータスコードを設定し、それ以外の場合は何もしません。
