multipartFormData
  • パーサーが追加で必要

要件：multipart/form-data関連

処理の流れ

ヘッダー解析　if (headerに　Content-Type: multipart/form-data　があったら)

ex. Content-Type: multipart/form-data; boundary=--<バウンダリ文字列>\r\n
  • バウンダリをsubstr()で格納する

複数のパート解析　次にContent-Dispositionを探す

ex. Content-Disposition: form-data; name="<1つ目の名前>"\r\n

getlineしながら、

  • データ毎に入れ物必要一つ目のデータ
    • 複数データがある場合がある
  • もし、if(.find() =="filename")
    • ファイルがある証拠
  • 空行の前はテキスト、後はバイナリ

data

  • 複数の"データ"を入れるvector<struct>
  • 一つ一つの構造体は↓
    struct ContentDisposition{
    　map<string, stirng> header; // 複数行のテキスト
    　std::vector<char> body; //写真やファイルのバイナリ
    }

↓具体的には、これをレキサー＆パーサー　　

  • sample
    --<バウンダリ文字列>\r\n
    Content-Disposition: form-data; name="<1つ目の名前>"\r\n
    \r\n
    <string 1つ目のデータ>
    --<バウンダリ文字列>\r\n
    Content-Disposition: form-data; name="<2つ目名前>"; filename="<ファイル名>"\r\n
    Content-Type: image/jpeg\r\n
    \r\n
    < vector<char> 画像のバイナリ >
    --<バウンダリ文字列>--\r\n

headerで定義したバウンダリ文字列でデータの区切りを表す。
バウンダリ文字列は–から始めないといけない制約がある。
すべての改行は\r
にする。(この改行の方法をCRLFと呼ぶらしい)
各データの先頭ではContent-Dispositionを指定していて、これがそのデータの情報を表している。あとは必要に応じてContent-Typeなどを指定する。
区切りの最後にもバウンダリ文字列を追加を追加する必要がある。この場合はバウンダリ文字列の最後にも–をつけないといけない。
