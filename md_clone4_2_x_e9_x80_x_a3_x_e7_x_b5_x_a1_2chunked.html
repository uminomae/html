chunked
  • NetworkService::とhttp双方に関わる範囲

要件Transfer-Encoding: chunked

Network側で登録

初回接続時 サーバpollのfdsのとこで

  • accept()の後にバッファmapのソケットをinit

http側で削除

  • parse headerで分岐
    • if Transfer-Encoding: chunked
  • parse body
    • recv()の後、ソケット毎にデータを格納
    • size==0ならmap のkeyソケットを削除（close()する箇所探して同時に）
    • 長時間経った、または、エラーが発生したら、close()と同時にバッファクリア
  • memo 他に格納する変数 チャンクサイズ
    CRLF
    連結

データ構造

  • class recvBufferにmap<int clientSocket, std::stirng buffer>　を独立して。
  • インスタンスの受け取り方はコンストラクタで注入するのが統一感あって良さそう

4.1.3. Decoding Chunked
A process for decoding the chunked transfer coding can be represented
in pseudo-code as:
length := 0
read chunk-size, chunk-ext (if any), and CRLF
while (chunk-size > 0) {
read chunk-data and CRLF
append chunk-data to decoded-body
length := length + chunk-size
read chunk-size, chunk-ext (if any), and CRLF
}
read trailer field
while (trailer field is not empty) {
if (trailer field is allowed to be sent in a trailer) {
append trailer field to existing header fields
}
read trailer-field
}
Content-Length := length
Remove "chunked" from Transfer-Encoding
Remove Trailer from existing header fields
